# Quest Control Console

Локальное адаптивное веб-приложение для оператора детского квеста в одной комнате.

## Что внутри

- единый главный пульт с крупными кнопками
- сценарный и ручной режимы
- экран сцен с быстрым переходом
- саундборд с play/stop и статусом воспроизведения
- панель света и эффектов
- подсказки по сценам и ручной ввод
- текстовый сценарий-шпаргалка
- учёт коктейлей после квеста
- журнал действий оператора
- mock-integration handlers под будущие реальные системы

## Структура

- `index.html` — точка входа
- `styles.css` — тёмная адаптивная тема
- `src/app.js` — логика интерфейса, action handlers, журнал
- `src/data-fallback.js` — встроенный демо-резерв на случай запуска без сервера
- `data/scenes.json` — сцены
- `data/sounds.json` — звуки и mp3-пути
- `data/effects.json` — эффекты и handlers
- `data/hints.json` — универсальные и сценические подсказки
- `data/cocktails.json` — карточки коктейлей
- `data/scenario.json` — текстовая шпаргалка
- `assets/audio/` — папка для локальных mp3

## Как запустить локально

### Вариант 1

Откройте `index.html` в браузере. Приложение запустится с резервными встроенными данными.

### Вариант 2

Запустите любой простой статический сервер и откройте корень проекта в браузере. Тогда приложение будет читать данные прямо из `data/*.json`, и сценарий можно будет менять без правок кода.

Подойдут:

- VS Code Live Server
- любой локальный http-server
- встроенный сервер вашей IDE

## Аудио

Положите mp3-файлы в `assets/audio/` с именами из `assets/audio/README.txt`.

Если файла нет, интерфейс не ломается, а пишет в журнал, что звук не найден.

## Дальнейшая интеграция

В `src/app.js` уже подготовлены отдельные handlers:

- `playSound(soundId)`
- `stopSound(soundId)`
- `closeCurtains()`
- `openCurtains()`
- `toggleFan()`
- `setChandelier(...)`
- `executeEffect(effectId)`

Сейчас они работают как mock-слой и пишут статусы в журнал. Позже сюда можно подключить реальные API, WebSocket, Home Assistant, Arduino, DMX-контроллер или любой другой контур управления.
